# План актуализации документации для установки и тестирования

## Цель

Создать комплексную документацию для установки проекта с нуля на отдельном компьютере (в т.ч. корпоративном), проведения полного цикла тестирования и возврата результатов разработчику.

## Текущая ситуация

Документация существует, но разрознена по нескольким файлам:

- `/README.md` - основной README с Quick Start
- `/digest-core/README.md` - подробная документация модуля
- `/docs/installation/INSTALL.md` - руководство по установке
- `/docs/installation/QUICK_START.md` - быстрый старт
- `/digest-core/docs/testing/MANUAL_TESTING_CHECKLIST.md` - чек-лист тестирования
- `/digest-core/docs/testing/SEND_RESULTS.md` - отправка результатов

**Проблемы:**

- Нет единого линейного потока "установка → тестирование → возврат результатов"
- Частичное дублирование информации
- Специфика корпоративных ноутбуков упоминается, но не систематизирована
- Нет единой точки входа для тестировщика

## Этапы работы

### 1. Создание нового End-to-End Testing Guide

**Файл:** `/docs/testing/E2E_TESTING_GUIDE.md`

Содержание:

- **Введение:** Для кого этот гайд, что будет установлено
- **Требования:** Минимальные требования к компьютеру, права доступа
- **Шаг 1: Подготовка окружения**
- Установка Git, Python 3.11+
- Специфика корпоративных ноутбуков (ограничения прав)
- Настройка прокси (если нужно)
- **Шаг 2: Клонирование и установка**
- Использование скрипта install_interactive.sh
- Альтернатива: ручная установка
- Проверка установки
- **Шаг 3: Настройка конфигурации**
- Получение учетных данных EWS
- Получение токена LLM
- Настройка переменных окружения
- Проверка подключения
- **Шаг 4: Smoke-тестирование**
- Запуск dry-run режима
- Проверка подключения к EWS
- Проверка нормализации данных
- **Шаг 5: Полное тестирование**
- Запуск полного цикла с test_run.sh
- Проверка результатов
- Метрики и логи
- **Шаг 6: Сбор диагностики**
- Автоматический сбор через collect_diagnostics.sh
- Проверка содержимого архива
- Безопасность (удаление секретов)
- **Шаг 7: Возврат результатов**
- Способы передачи (email, файловая система, USB)
- Шаблон отчета
- Что включить в отчет
- **Troubleshooting:** Типичные проблемы на корпоративных ноутбуках
- **FAQ:** Частые вопросы
